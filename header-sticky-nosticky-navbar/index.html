Sample Mobile Sticky Header Ad Implementation

The header ad has a few configuration parameters that you can tweak.

  • CSS variables: The sticky-ad.css file contains some variables that may need adjustments according to your ad layout. The --header-ad-height may be 250px or only 50px. The --navbar-height may differ for your layout.
  • The --header-ad-close-button can be set to none if you don't want a close button or you can remove the button tag entirely.
  • The fadeOutClassname. By default we use header-ad--fadeOut. This class gets applied, when the slot should fade out.
  • The selector for the DOM node that should be observed for visibility, which triggers the header ad fade out. We recommend using the first content position. You can however turn this feature off and the sticky header will always be visible. We only recommend this if you have no mobile sticky ad on your page.

The sticky implementation in this example is only for mobile.

CSS

Similar to the Google Anchor Ad Top The general idea is that an Intersection Observer watches for a certain element, usually the first content ad slot, to be visible and then adds a fade-out CSS class, which transitions the slot out of viewport. In order to make this work the space needs to be received. This example uses padding top to achieve this. The ad slot it self floats on top of this empty space.
